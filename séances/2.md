# Du cycle en V aux méthodes agiles

## Objectif
Pendant cette séance, vous allez :
- comprendre les limites du cycle en V,
- découvrir les principes de l’agilité,
- comparer deux approches de gestion de projet.

---

## 1. Rappel : le cycle en V

Le cycle en V est une méthode de gestion de projet **séquentielle** :
- les étapes s’enchaînent dans un ordre précis,
- les tests sont prévus dès la phase de conception,
- les validations sont réalisées en fin de cycle.

Cette méthode est efficace lorsque :
- le besoin est bien défini dès le départ,
- le projet est stable,
- les changements sont rares.

---

## 2. Limites du cycle en V

Le cycle en V présente plusieurs limites :

- Difficulté à intégrer les changements en cours de projet
- Retour utilisateur tardif
- Risque de découvrir des erreurs importantes en fin de projet
- Documentation lourde
- Peu de flexibilité

Ces limites deviennent problématiques lorsque :
- le besoin évolue,
- le client n’est pas certain de ses attentes,
- le contexte change rapidement.

---

## 3. Origine des méthodes agiles

Les méthodes agiles sont apparues pour répondre à ces limites.

Elles reposent sur une idée simple :
> S’adapter au changement plutôt que le subir.

Les méthodes agiles privilégient :
- l’adaptation,
- la collaboration,
- les retours fréquents,
- la livraison progressive.

---

## 4. Principes fondamentaux de l’agilité

Les méthodes agiles reposent sur plusieurs principes clés :

- Le client est impliqué tout au long du projet
- Le produit est livré par petites parties
- Les retours sont réguliers
- Les changements sont acceptés
- La communication est centrale
- Le projet évolue par itérations courtes

---

## 5. Fonctionnement général d’un projet agile

Un projet agile est organisé en cycles courts appelés **itérations** ou **sprints**.

À chaque itération :
- une partie du produit est développée,
- testée,
- livrée,
- puis améliorée grâce aux retours.

Il n’y a pas de validation unique en fin de projet, mais des validations continues.

---

## 6. Comparaison cycle en V / méthodes agiles

| Critère | Cycle en V | Méthodes agiles |
|------|-----------|----------------|
| Organisation | Séquentielle | Itérative |
| Flexibilité | Faible | Élevée |
| Gestion des changements | Difficile | Naturelle |
| Livraison | En fin de projet | Progressive |
| Rôle du client | Peu impliqué | Très impliqué |
| Documentation | Très importante | Allégée |
