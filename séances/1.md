# Découverte du cycle en V

## Objectif
Pendant cette séance, vous allez :
- découvrir le cycle en V,
- comprendre ses étapes,
- l’appliquer à un projet simple.

Vous devez lire attentivement ce document et être capable d’en extraire les informations importantes.

---

## 1. Qu’est-ce que le cycle en V ?

Le **cycle en V** est une méthode de gestion de projet dite **séquentielle**.

Cela signifie que les étapes du projet se déroulent dans un ordre précis :
- on ne passe à l’étape suivante que lorsque la précédente est terminée.

Le cycle en V tient son nom de sa forme :
- la branche gauche correspond aux phases de **conception**,
- la branche droite correspond aux phases de **validation**.

---

## 2. Principe fondamental

Le principe clé du cycle en V est le suivant :

> À chaque phase de conception correspond une phase de test ou de validation.

Les tests ne sont donc pas réalisés au hasard, mais sont préparés dès le début du projet.

---

## 3. Les étapes du cycle en V

### 3.1 Expression des besoins
Cette étape consiste à comprendre ce que veut le client :
- ses attentes,
- ses objectifs,
- ses contraintes.

**Livrable principal :**
- Cahier des charges

---

### 3.2 Spécifications fonctionnelles
Cette étape décrit ce que le système doit faire :
- fonctionnalités attendues,
- comportement général.

**Livrable principal :**
- Spécifications fonctionnelles

---

### 3.3 Spécifications techniques
Cette étape définit comment le système va être réalisé :
- choix techniques,
- architecture,
- outils utilisés.

**Livrable principal :**
- Dossier de conception technique

---

### 3.4 Développement
Cette étape correspond à la réalisation du projet :
- écriture du code,
- création du produit.

**Livrables :**
- Code source
- Application fonctionnelle

---

### 3.5 Tests unitaires
Les tests unitaires permettent de :
- tester chaque composant séparément,
- vérifier que chaque partie fonctionne correctement.

---

### 3.6 Tests d’intégration
Les tests d’intégration permettent de :
- vérifier que les composants fonctionnent ensemble,
- détecter les erreurs d’interaction.

---

### 3.7 Tests de validation (recette)
Cette étape consiste à vérifier que le produit :
- correspond au cahier des charges,
- répond au besoin du client.

**Livrable :**
- Procès-verbal de recette

---

## 4. Correspondance conception / validation

| Conception | Validation |
|-----------|------------|
| Expression des besoins | Tests de validation |
| Spécifications fonctionnelles | Tests d’intégration |
| Spécifications techniques | Tests unitaires |
